The full thought process

From a blank tenant to a signed-off inventory

Each act hands the next a more trustworthy artifact. Nothing irreversible happens until a human has seen it.

Setup — turn paper-reality into assignable work

Admin sets fiscal year, source of truth, locations, rules. The adapter pulls master data from SAP via OData/BAPI and maps it to the internal model. The president forms commissions and assigns lists.

Field — capture reality without a network

Member downloads lists into an encrypted local DB. Scan, manual lookup, or add a surplus candidate; set status, note, photo. Local rules fire instantly.network irrelevant

Sync — reconnect without loss or duplicates

Events carry UUIDs; the server treats them idempotently and returns a cursor. The device clears only acknowledged events; a dropped batch resumes exactly where it stopped.no data loss

Reconcile — compare two versions of the truth

The engine compares physical (events) vs. book (master) and classifies each asset into one disposition. Clean matches auto-close; financial cases route to a gate.

Review — humans resolve the exceptions

Presidents clear the Conflict Inbox; accounting and auditors work financial exceptions. Separation of duties enforced; every decision written to the append-only audit log.

Return & report — close the loop

Approved metadata updates master data; approved financials become posting candidates a human commits — never auto-posted. Prescribed reports export to xlsx/docx/pdf.

The AI & analytics layer

Makes the inventory data askable and watchable — but it never decides. Every number traces to an auditable record; every action still passes a human gate.

How natural-language query stays safe

It is not “send the database to an LLM.” The model proposes; a validator disposes.

Validator checks the query
